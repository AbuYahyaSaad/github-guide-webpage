Setting Up Git & GitHub

1. Install Git

Download and install Git from the official website for your operating system.

2. Configure Git

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

3. Create GitHub Account

  • Go to github.com
  • Click "Sign up"
  • Follow the registration process
  • Verify your email

4. Generate SSH Key (Optional but Recommended)

ssh-keygen -t ed25519 -C "your.email@example.com"

Essential Git Commands

Basic Commands

git init # Initialize a new repository
git clone [url] # Clone a repository
git status # Check status
git add . # Stage all changes
git commit -m "message" # Commit changes
git push # Push to remote
git pull # Pull from remote

Branch Commands

git branch # List branches
git branch [name] # Create new branch
git checkout [name] # Switch branches
git checkout -b [name] # Create and switch
git merge [branch] # Merge branch
git branch -d [name] # Delete branch

Advanced Commands

git log --oneline # View commit history
git reset --hard HEAD~1 # Undo last commit
git stash # Temporarily save changes
git rebase [branch] # Rebase current branch
git cherry-pick [commit] # Apply specific commit

GitHub Workflow

Standard Git Flow

Clone/Init ‚Üí Make Changes ‚Üí Stage ‚Üí Commit ‚Üí Push

Step-by-Step Process

  1. Clone Repository: git clone https://github.com/user/repo.git
  2. Create Branch: git checkout -b feature-branch
  3. Make Changes: Edit your files
  4. Stage Changes: git add .
  5. Commit: git commit -m "Add new feature"
  6. Push: git push origin feature-branch
  7. Create Pull Request: On GitHub website

Collaboration Features

Pull Requests

Pull requests let you tell others about changes you've pushed to a branch in a repository.

  • Create descriptive titles and descriptions
  • Reference related issues
  • Request reviews from team members
  • Respond to feedback and make changes

Issues

Use issues to track ideas, enhancements, tasks, or bugs.

  • Create detailed issue descriptions
  • Use labels to categorize
  • Assign to team members
  • Link to pull requests

Code Review

Review Changes

Examine code modifications line by line

Leave Comments

Provide feedback on specific lines

Approve/Request Changes

Give formal approval or ask for modifications

Advanced GitHub Features

name: CI Pipeline on: [push, pull_request] jobs: test: runs-on: ubuntu-latest steps: - uses: actions/checkout@v2 - name: Run tests run: npm test

GitHub Pages

Host static websites directly from your repository.

  • Enable in repository settings
  • Choose source branch
  • Access at: username.github.io/repository
